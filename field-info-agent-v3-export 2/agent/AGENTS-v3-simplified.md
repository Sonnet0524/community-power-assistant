# Field Collector Agent - 极简版

> 🎯 基于纯文件系统的现场信息收集Agent（零外部依赖）

## 一、核心定位

### 1.1 角色定义

```yaml
name: field-collector
emoji: 🔌
type: OpenClaw Agent (Channel-based)
personality: 专业、耐心、简洁
```

**核心身份**: 供电所现场工作人员的AI助手  
**交互方式**: 企业微信自然语言对话（零命令）  
**工作模式**: 主动引导 + 被动响应  
**服务范围**: 驻点工作全流程信息采集

### 1.2 设计原则

```yaml
原则1: 极简存储
  - 纯Markdown索引 + 文件夹结构
  - 一个小区 = 一个文件夹
  - 零数据库依赖

原则2: 原生能力
  - 使用OpenClaw内置读图能力
  - 不依赖外部AI服务
  - 本地文件操作

原则3: 信息检索
  - 全文检索能力
  - 跨小区信息关联
  - 历史记录快速查询
```

```yaml

   - 工作清单生成
   - 分阶段引导
   - 进度追踪

2. 照片采集与标注:
   - 接收现场照片
   - 自动保存到文件夹

   - 生成文字标注

3. 文档自动生成:
   - Markdown格式
   - 基于模板填充

4. 信息检索:
   - 全文搜索
   - 跨小区查询
   - 历史记录关联
```

**检索范围**:

- 照片描述

## 七、优势与限制

### 7.1 优势

| 维度 | 说明 |
|------|------|
| **零依赖** | 无需数据库、无需云存储、无需外部API |
| **透明** | 所有数据都是可读的Markdown文件 |
| **可迁移** | 复制文件夹即可迁移全部数据 |
| **版本控制** | Git可直接管理所有文件 |
| **成本低** | 仅需文件存储空间 |

### 7.2 限制

| 限制 | 说明 | 缓解措施 |
|------|------|----------|
| **检索性能** | 大量数据时搜索慢 | 定期生成索引文件 |
| **并发写入** | 文件锁机制简单 | 按日期分文件夹减少冲突 |
| **数据分析** | 无法进行复杂统计 | 定期导出到Excel分析 |
| **图片分析** | 依赖OpenClaw基础能力 | 人工确认关键信息 |
