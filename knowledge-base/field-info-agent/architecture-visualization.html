业务整体说明

Field Info Agent（现场信息收集智能体）是面向供电所驻点工作场景的智能化辅助系统。 通过企业微信自然语言交互，实现驻点工作的全流程数字化管理。

🎯

核心目标

提升驻点工作效率，规范信息采集流程，沉淀社区知识资产

👥

目标用户

供电所客户经理、抢修人员、驻点工作人员

💡

交互方式

零命令自然语言，企业微信消息/照片/位置

业务关键内容模块

1

信息预准备

工作启动阶段

  • ✓ 自然语言识别驻点意图（"去阳光社区驻点"）
  • ✓ 查询社区历史驻点记录和重点客户
  • ✓ 动态生成工作清单（配电房/走访/应急）
  • ✓ 初始化Session会话状态
2

现场指引

信息采集阶段

  • ✓ 多轮对话引导现场采集流程
  • ✓ 接收并保存照片到对象存储
  • ✓ 记录位置信息和文字描述
  • ✓ 实时触发AI照片分析
3

内容分析审核

AI智能处理阶段

  • ✓ KIMI多模态识别电力设备类型
  • ✓ 自动检测设备缺陷（漏油/锈蚀/过热）
  • ✓ 评估设备状态（正常/注意/异常/危险）
  • ✓ 批量分析生成整体评估报告
4

报告生成

成果输出阶段

  • ✓ 汇总采集数据和分析结果
  • ✓ Word模板自动填充数据
  • ✓ 自动生成驻点工作记录表
  • ✓ 归档到知识库更新社区信息

核心流程

1

启动工作

用户发送"去XX社区驻点"，Agent创建Session，查询历史记录

2

现场采集

用户拍摄照片并发送，Agent保存到MinIO，触发AI分析

3

AI分析

KIMI多模态分析照片，识别设备和缺陷，结果存入PostgreSQL

4

生成报告

汇总采集数据，填充Word模板，生成驻点工作记录表

数据架构

🗄️ PostgreSQL

field_sessions
会话主表，记录驻点工作基本信息
photo_analysis
AI分析结果，设备识别和缺陷
community_info
社区信息，历史驻点记录
generated_docs
生成文档记录和版本

📦 MinIO

/photos/
原始照片，按日期/社区组织
/documents/
生成的Word文档
/templates/
Word文档模板
/thumbnails/
照片缩略图

⚡ Redis

session:{userId}
活跃会话状态，TTL 2小时
lock:{userId}
并发控制锁
temp:{key}
临时数据缓存
rate:{userId}
API限流计数
