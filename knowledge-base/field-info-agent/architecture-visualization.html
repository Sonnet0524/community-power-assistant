Agent 框架

协议 WebSocket
AI

AI 模型

大模型 KIMI K2.5
能力 多模态
功能 意图+视觉
DB

数据存储

关系型 PostgreSQL 16
对象存储 MinIO
缓存 Redis 7

核心流程

1

启动工作

用户发送"去XX社区驻点"，Agent创建Session，查询历史记录

2

现场采集

用户拍摄照片并发送，Agent保存到MinIO，触发AI分析

3

AI分析

KIMI多模态分析照片，识别设备和缺陷，结果存入PostgreSQL

4

生成报告

汇总采集数据，填充Word模板，生成驻点工作记录表

数据架构

🗄️ PostgreSQL

field_sessions
会话主表，记录驻点工作基本信息
photo_analysis
AI分析结果，设备识别和缺陷
community_info
社区信息，历史驻点记录
generated_docs
生成文档记录和版本

📦 MinIO

/photos/
原始照片，按日期/社区组织
/documents/
生成的Word文档
/templates/
Word文档模板
/thumbnails/
照片缩略图

⚡ Redis

session:{userId}
活跃会话状态，TTL 2小时
lock:{userId}
并发控制锁
temp:{key}
临时数据缓存
rate:{userId}
API限流计数
