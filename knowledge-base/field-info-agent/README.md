# Field Info Agent - 现场信息收集智能体

## 项目概述

**项目名称**: Field Info Agent（现场信息收集智能体）  
**技术栈**: OpenClaw + 企业微信 + KIMI 2.5 + MinIO + PostgreSQL
**可行性**: ✅ 完全可行
**状态**: 🟢 OpenClaw实现结构已完成，等待开发启动

---

## 核心价值

> **让现场人员"随手拍、打字说"，Agent自动完成信息结构化、照片智能分析、文档生成、知识沉淀**

---

## 核心功能

### 1️⃣ 驻点工作引导（StationWorkGuide）
- **工作启动**: 识别小区，加载历史，生成任务清单
- **配电房采集**: 文字记录位置，AI识图识别铭牌，批量照片智能分析
- **客户走访**: 推荐客户，文字记录反馈，满意度标记
- **应急采集**: 拍照记录接入点，自动生成指引

### 2️⃣ 照片智能分析（VisionAnalysis）⭐ 新功能
- **设备状态评估**: AI分析设备照片，判断运行状态
- **隐患自动发现**: 从照片中识别安全隐患
- **批量照片分析**: 收集完毕后对所有照片综合分析
- **智能报告生成**: 基于照片内容生成分析报告

### 3️⃣ 文档自动生成（AutoDocGeneration）
- **小区供电简报**: 自动汇总小区供电情况（含照片分析结果）
- **应急发电指引**: 提取应急信息，生成指引文档
- **驻点工作总结**: 汇总工作成果、照片分析和问题
- **优质服务简报**: 统计客户满意度和服务质量

### 4️⃣ 应急处置（EmergencyGuide）
- **应急启动**: 快速响应，加载应急资料
- **敏感客户关怀**: 自动识别敏感客户，推送清单
- **应急方案推送**: 调取应急指引，指导现场操作
- **过程记录**: 记录处理节点，生成报告

## 关键技术决策（v2.1 更新）

### 1. 企业微信集成方案
**决策**: 自建应用（一对一服务）+ 群机器人（通知推送）

**理由**:
- 自建应用支持完整消息接收（图片、文字、位置）
- 支持长流程多轮对话
- 群机器人用于群发通知

**更新（v2.1）**:
- ~~语音消息~~ → **仅文字消息**
- 用户使用语音输入法自行转文字后发送

### 2. ~~语音识别~~ → 文字输入
**决策**: **仅接收文字输入**，不提供语音识别服务

**理由**:
- 简化技术架构
- 现代手机语音输入法成熟准确
- 降低开发复杂度和成本
- 用户已习惯语音输入法

### 3. ~~PaddleOCR~~ → KIMI K2.5 多模态识图 ⭐
**决策**: **使用 KIMI K2.5 进行图像理解**

**理由**:
- 国产模型，合规性好
- 中文场景优化，理解能力强
- 不仅识别文字，还能理解场景和设备状态
- 可发现隐患（如"变压器漏油"、"设备锈蚀"）
- 无需部署OCR服务
- 自然语言交互，更智能

**能力**:
- 单张照片实时分析
- 批量照片综合分析
- 设备状态评估
- 安全隐患识别
- 长文本报告生成

### 4. 数据存储（v2.2更新）
**决策**: ~~WPS云文档~~ → **本地存储（MinIO + PostgreSQL）** ⭐

**理由（变更）**:
- ~~WPS权限控制不够灵活~~ → **本地存储完全自主控制**
- ~~按量计费可能较高~~ → **固定成本，约节约¥300/月**
- 数据主权：本地存储符合数据安全要求
- 站级隔离：可按供电所完全隔离数据

**存储架构**:
- **PostgreSQL**: 结构化数据（会话、分析结果、知识库）
- **MinIO**: 非结构化数据（照片、Word文档）
- **Redis**: 会话缓存
- **版本控制**: 所有数据版本化，历史可追溯

### 5. 新增：照片智能分析策略
**决策**: **实时轻量确认 + 完成后深度分析**

**策略**:
- **采集过程中**: 简单确认照片已收到
- **采集完成后**: 批量深度分析所有照片
- **生成报告时**: 整合分析结果到文档
