# 设计变更摘要（v2.1）

**日期**: 2026-03-17  
**变更类型**: 需求调整与技术优化  
**变更原因**: 简化架构、提升智能化水平

---

## 三大变更

### 1️⃣ 语音 → 文字

**变更前**:
- 企业微信接收语音（AMR）
- 百度语音识别转文字
- 支持四川话识别

**变更后**:
- ✅ **仅接收文字消息**
- 用户使用语音输入法自行转文字
- Agent只处理文字

**收益**:
- 技术架构简化
- 降低开发复杂度
- 节省成本¥150/月

---

### 2️⃣ OCR → KIMI K2.5 多模态识图

**变更前**:
- 部署PaddleOCR本地服务
- 结构化识别铭牌信息
- 仅识别文字

**变更后**:
- ✅ **使用KIMI K2.5多模态模型**
- AI理解图片内容（文字+场景+状态）
- 可发现隐患（如"变压器漏油"）
- 国产模型，合规性好

**收益**:
- 识别能力更强
- 无需维护OCR服务
- 自然语言交互
- 新增批量照片分析能力

**成本**:
- 新增：¥300-800/月（LLM API）
- 节省：¥0（无需OCR服务器）

---

### 3️⃣ 新增：批量照片智能分析

**新功能**:
- 收集大量现场照片后批量分析
- 设备状态自动评估
- 隐患自动发现
- 生成分析报告

**应用场景**:
- 配电房采集（5-10张照片）
- 设备巡检照片分析
- 安全隐患识别

---

## 技术架构对比

### v2.0 架构
```
用户层: 语音 + 拍照 + 文字
   ↓
Tools: WPS API + 百度语音 + PaddleOCR + WeCom API
```

### v2.1 架构
```
用户层: 拍照 + 文字（用户自转语音）
   ↓
Tools: WPS API + 多模态LLM ⭐ + WeCom API
```

**关键变化**:
- 移除：百度语音、PaddleOCR
- 新增：KIMI K2.5 多模态识图
